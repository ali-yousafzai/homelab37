**What is a Home Lab?**

That's the question I would've had up until only a few months ago. My home lab will be a computer environment with multiple components that are able to communicate with one another. It's essentially a personal small scale server.

When I used to work at a datacenter, I would get questions as to what a server even is. My answer would be 'Basically, a really, really powerful computer that communicates with the computers we use at home to make them more useful. They're mainly used for data storage, virtualization, web hosting, streaming, and nowadays, AI teaching and usage.' Perhaps my answer wasn't quite as graceful as that in the moment, but I gave them the gist of it. All in all, I can do all of those things with a personal home lab, and this repository is where I will document that.

For my personal use case, this home lab will be used as a learning exercise. I want to strengthen my experience working with the Linux OS, networking architecture, and virtualization/VMs. Cybersecurity wise, I want hands on experience working with endpoint detection agents, SIEM log management tools, identity/certificate management, and simulated attacks. This kind of setup gives me a chance to learn without having to worry about potentially putting an enterprise environment at risk. The best way to learn is to make mistakes, and I'm ready to make them.
